# 上下文感知补全功能实现说明

## 功能说明
实现了根据 XML 上下文智能显示补全项的功能：
1. **标签名位置**（如 `<|`）：只显示所有 SilkyUI 类名
2. **属性名位置**（如 `<Button |`）：只显示当前标签对应类的所有属性
3. **属性值位置**（如 `<Button Visibility="|`）：只显示当前属性对应的枚举值
4. 如果属性不是枚举类型或没有枚举值，属性值位置不会显示任何补全项

## 实现改动
### 1. 添加了 XML 上下文相关定义
- `XmlContextType` 枚举：定义了三种上下文类型（标签名、属性名、属性值）
- `XmlContext` 类：保存上下文信息（当前上下文类型、当前标签名、当前属性名）

### 2. 添加了上下文分析方法
`AnalyzeXmlContext` 方法：
- 分析当前光标位置的 XML 结构
- 判断当前所处的上下文类型
- 提取当前标签名和属性名

### 3. 修改了补全逻辑
`AugmentCompletionSession` 方法：
- 根据上下文类型过滤补全项
- 只显示当前上下文相关的补全内容
- 没有匹配的补全项时不显示补全弹窗

## 使用示例
```xml
<!-- 输入 < 时，只显示类名补全 -->
<Button 

<!-- 在标签内输入空格时，只显示 Button 类的属性补全 -->
<Button Visibility=

<!-- 在属性的引号内时，只显示 Visibility 属性的枚举值 -->
<Button Visibility="Visible"
```

## 注意事项
- 目前仅支持双引号的属性值，单引号暂不支持
- 上下文分析基于当前行，跨多行的标签可能无法正确识别
- 枚举值只有在属性类型确实是枚举时才会显示
